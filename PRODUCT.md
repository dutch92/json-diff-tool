# Product

## Register

product

## Users

Developers comparing JSON payloads during debugging, API review, fixture updates, and regression checks. They arrive focused, often with large or unfamiliar data, and need to paste, inspect, edit, and navigate differences without ceremony.

## Product Purpose

JSON Diff Tool compares two JSON documents side by side, renders structured trees, highlights added, removed, and changed values, and lets users move through differences quickly. Success means the user can understand what changed in a payload with minimal setup and high confidence.

## Brand Personality

Precise, compact, dependable. The interface should feel like a serious developer workbench: calm under pressure, direct in language, and visually quiet enough for long sessions.

## Anti-references

Do not become a glossy SaaS dashboard, a playful JSON tutorial, a docs-heavy teaching tool, or a heavy IDE clone. Avoid decorative gradients, tutorial panels, marketing-style hero composition, and UI chrome that competes with the JSON.

## Design Principles

- Preserve the split-pane mental model.
- Keep repeated actions close to the panes they affect.
- Make state visible without turning it into instruction.
- Prioritize readable code surfaces over decorative layout.
- Treat density as a feature when it improves comparison speed.

## Accessibility & Inclusion

Target WCAG AA contrast for text, controls, focus states, and diff highlights. Use shape and labels in addition to color for status. Keep keyboard focus visible in the editor, tree, toolbar, theme menu, and diff navigation.
