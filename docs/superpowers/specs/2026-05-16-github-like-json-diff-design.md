# GitHub-Like JSON Diff Viewer Design

## Goal

Переработать режим просмотра JSON так, чтобы он ощущался как просмотр diff в GitHub: каждая видимая строка живет в плотной строковой сетке, слева показан номер строки, рядом отдельная колонка маркера изменения, а содержимое JSON остается читаемым и сворачиваемым.

## Selected Approach

Берем вариант A из визуального обсуждения: построчный diff-блок внутри каждого JSON viewer.

Текущий древовидный renderer сохраняется концептуально, но его DOM и CSS меняются на GitHub-like rows:

- колонка номера строки;
- колонка diff-маркера (`+`, `-`, `~`) с цветной подложкой для измененных путей;
- колонка JSON-содержимого;
- стрелка сворачивания внутри строки-контейнера;
- подсветка активного diff на всю строку.

Номера строк являются номерами отображаемых строк, а не номерами исходного введенного текста. Это важно, потому что viewer уже парсит JSON, сортирует ключи объектов и отображает структуру как дерево.

## User Experience

В режиме `Показать дерево` пользователь видит два JSON viewer рядом. Каждый viewer похож на компактный diff-файл:

- строки идут одна под другой с моноширинным текстом;
- неизмененные строки имеют нейтральную колонку номера;
- добавленные, удаленные и измененные строки получают цвет фона и marker rail;
- контейнеры объектов и массивов раскрываются и сворачиваются через стрелку;
- свернутый контейнер показывает короткую форму вида `"profile": { ... }` или `"items": [ ... ]`;
- примитивные значения по-прежнему редактируются кликом;
- переход по diff из верхней навигации прокручивает к активной строке и раскрывает ее предков.

Режим редактирования исходного JSON остается текстовым textarea без номеров строк в рамках этой итерации.

## Component Changes

Основная работа находится в `src/components/JsonViewer/JsonTree.tsx` и `src/components/JsonViewer/JsonViewer.css`.

`JsonTree` добавит модель видимых строк:

- каждый `JsonNode` получает отображаемый номер строки;
- opening line, primitive line и closing line рендерятся через общий row layout;
- для раскрытых контейнеров closing brace/bracket получает собственный номер строки;
- для свернутых контейнеров children и closing line не рендерятся;
- `collapsedPaths` и automatic `expandedPaths` для активного diff сохраняются.

Для минимального риска счетчик строк можно реализовать во время рендера через mutable counter внутри `JsonTree`, потому что он нужен только для визуальной нумерации текущего render pass.

## Styling

CSS переводится с текущего `border-left` rail на grid layout:

- `.json-node` или новый row-класс использует `grid-template-columns` для номера, маркера и содержимого;
- номерная колонка имеет приглушенный фон, фиксированную ширину и выравнивание вправо;
- marker column показывает `+`, `-`, `~` только для строк с diff-статусом;
- цветовые состояния `added`, `removed`, `changed`, `active` применяются к полной строке;
- вложенность переносится в padding/margin контентной колонки, чтобы номера строк оставались в одной вертикальной колонке;
- горизонтальный overflow сохраняется для длинных JSON-значений.

## Accessibility

Кнопки сворачивания сохраняют `aria-expanded`. Редактируемые значения сохраняют текущие `aria-label`. Номера строк помечаются как декоративные или получают нейтральный текст без интерактивности, чтобы не засорять keyboard flow.

## Tests

Обновить существующий `tests/diffMarkerLayout.test.mjs`, потому что он сейчас проверяет старую colored rail модель. Новый тест должен подтверждать:

- наличие grid-колонок для номера строки и marker column;
- наличие классов/стилей для line number column;
- marker backgrounds для `added`, `removed`, `changed`;
- active row styling на уровне строки.

Существующие тесты diff/user message/edit value не должны меняться, если поведение данных не затрагивается.

## Out Of Scope

- unified diff с двумя колонками старых/новых номеров;
- line numbers в textarea-редакторе;
- синхронный scroll между левым и правым viewer;
- сохранение collapsed state между перезагрузками;
- изменение алгоритма вычисления diff.
